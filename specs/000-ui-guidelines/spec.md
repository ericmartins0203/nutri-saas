# UI Guidelines

## Goal
Keep the product simple, educational and easy to use for independent nutritionists and their clients.

## Principles

- Prioritize clarity over density.
- Use simple language instead of clinical jargon when client-facing.
- Make the client area mobile-first.
- Use cards and short sections instead of long tables for client screens.
- Show progress beyond weight when possible.
- Keep nutritionist workflows fast and low-friction.
- Avoid making the interface feel like a hospital system.

## Nutritionist area

- Dashboard should answer: who needs attention today?
- Client profile should centralize diet, check-ins, progress and finance.
- Forms should be short and progressive.
- Destructive actions should ask for confirmation.

## Client area

- Home should show objective, next action and latest progress.
- Diet page should be easy to read on mobile.
- Check-in should take only a few minutes.
- Progress should feel motivating, not punitive.

## Visual direction

- Clean layout.
- Calm colors.
- Rounded cards.
- Clear spacing.
- Accessible text contrast.
